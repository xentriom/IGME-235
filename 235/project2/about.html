About the Pokédex

The Pokedex has been meticulously developed as a distinguished project for academic purposes created in 3 weeks. It seamlessly integrates with the PokiAPI, leveraging its extensive repository of Pokémon data to offer users a refined and comprehensive experience. The objective is to create a site that provides basic yet insightful knowledge at a glance, catering to all Pokemon enthusiasts.

Account System

Users are able to create and register their own accounts to save various Pokemons. There is no limit to the amount of Pokemons you can save per account and each account has their own storage!

NOTE: Due to project requirement, the accounts are saved in local storage rather than utilizing a database like mySQL. Using local storage is NOT secure and recommened.

Showcase Display

View up to 32 Pokemons generated randomly or by pokedex number! Each are displayed with their name and an image of the Pokemon. Hovering over the Pokemon will outline the box they are in to give the user a clear indication. Lastly, the user can press on the Pokemon and it will generate an infographic and show that to the user.

Create Infographics

Generates an infographic showcasing various information regarding the searched / selected Pokemon. The infographic contains many informaion such as the Pokemon's name, type, height, weight, and more! Try it out and see for yourself!

HOW DID I MAKE IT?

Here are the steps I used to create the best Pokédex website!

Step 1: Gathering intel and creating mockups of the site you envisioned.

Step 2: Soak in the API, relax and take your final breaths as you start consuming the API.

Step 3: Javascript time! Just jump straight into it, start by making all the neccessary API calls and get all the data!

Step 4: You got the data, you got the vision, now start building! One div at a time, start layering each info onto one another.

Step 5: Putting it all together, cook up the final divs and wa-la, you got yourself a fully functional site, without css.

Step 6: Cascading Style Sheets... alas the time has come, when all but css is done. Design principles, color theory, "why is this div not centering???", and rwd - responsive web design.

And that is how I created the best Pokédex website ever.

THOUGHTS

My thought process was pretty easy at first, them abition took over and I ended up challenging myself to various things. Initially I wanted to create the infographic, display screen was never meant to be.

While working on the JavaScript, I realize I would be scrolling a lot due to various features I wanted to add, so I ended up breaking the files up by functionality. There is a file for each task, that way debugging is much simplier and I don't have to scroll through hundreds of line to find the function.

That thought of seperating each feature into it's own file ended up following to the css as well. Most of it is in 1 file but I split them up for various tasks and that made finding and editing the respective elements easier.

On the right, I have images of several iterations of my mockups. As you can see, I frequently took screenshots of my page and drew over them as I envisioned what I wanted the website to look like. None of the mockups had any color since I am really bad at picking out color, hence the very bland and simplistic color choices.
